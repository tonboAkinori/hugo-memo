-Wabsolute-value

より適切な標準関数が使用可能な場合に、引数の絶対値を計算する標準関数の呼び出しについて警告する。 たとえば、abs（3.14）を呼び出すと警告される。 これは、double引数の絶対値を計算するために呼び出す適切な関数がfabsであるためです。

#include<stdio.h>
#include<stdlib.h>
#include<math.h>

int main(void)
{
  unsigned char tmp0 = abs(10.3);
  unsigned char tmp1 = abs(10);
  double tmp2 = fabs(10);
  double tmp3 = fabs(10.3);

  return 1;
}

sample.c:7:24: warning: using integer absolute value function ‘abs’ when argument is of floating point type ‘double’ [-Wabsolute-value]
    7 |   unsigned char tmp0 = abs(10.3);
sample.c:9:17: warning: using floating point absolute value function ‘fabs’ when argument is of integer type ‘int’ [-Wabsolute-value]
    9 |   double tmp2 = fabs(10);

-Waddress

メモリアドレスの疑わしい使用について警告します。 これには、void func（void）などの条件式で関数のアドレスを使用することが含まれます。 if（func）、およびif（x == “abc”）などの文字列リテラルのメモリアドレスとの比較。 このような使用法は通常、プログラマーのエラーを示します。 関数のアドレスは常にtrueと評価されるため、条件付きでの使用は通常、プログラマーが関数呼び出しの括弧を忘れたことを示します。 また、文字列リテラルとの比較は、不特定の動作をもたらし、Cで移植できないため、通常、プログラマーがstrcmpを使用することを意図していたことを示します。 この警告は-Wallによって有効になります。

#include<stdio.h>
void myfunc(void)
{
}

int main(void)
{
 if(myfunc)
   return 0;
 return 1;
}

sample.c:9:6: warning: the address of ‘myfunc’ will always evaluate as ‘true’ [-Waddress]
    9 |   if(myfunc)
      |
